# 서버 상태 모니터링 컨텍스트

## 1. 서버 상태 분류 기준
- 정상(Normal): CPU/메모리/디스크 사용률 70% 미만, 서비스 정상 동작
- 경고(Warning): CPU/메모리/디스크 사용률 70~89%, 서비스 응답 지연
- 심각(Critical): CPU/메모리/디스크 사용률 90% 이상, 서비스 중단/오류

## 2. 주요 모니터링 지표
- CPU 사용률 (%)
- 메모리 사용률 (%)
- 디스크 사용률 (%)
- 네트워크 지연(ms)
- 서비스 응답 속도(ms)
- 장애/오류 발생 내역

## 3. 이상 상황 예시
- CPU 사용률 90% 이상 지속
- 메모리 부족으로 인한 서비스 재시작
- 디스크 공간 95% 이상 사용
- 네트워크 지연 200ms 초과
- 서비스 다운/오류 다수 발생

## 4. 장애/이상 보고서 작성 가이드
- 장애 발생 시점, 영향받은 서버/서비스, 주요 지표 변화, 원인 추정, 조치 내역, 권장 대응 방안 포함

## 5. 모니터링/분석 도구
- 실시간 대시보드, 알림 시스템, 자동 분석(AI), 장애 이력 관리
